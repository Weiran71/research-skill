# 调研维度清单

> 此文件由用户维护。调研者必须**逐一对照**清单中的每个维度。
> - 适用的 → 必须覆盖
> - 不适用的 → 给出跳过原因 + 等价替代（如有），用户有权要求补上
> - 清单之外，调研者可根据具体对象自由发挥、补充更多维度

## 必选维度

### 1. 整体架构与流程
- 必须产出整体架构图或流程图
- 展示系统的分层、模块关系、数据流向
- 图表需清晰可读，使用中文标注

### 2. 设计理念与思想
- 这个产品/技术背后的核心设计哲学是什么
- 它为什么要这样设计，而不是其他方式
- 它独特的、区别于同类的思想内核

### 3. 核心/特色能力详解
- 不是每一家都有的，是它独有的设计和功能
- 每项核心能力需深度展开：关键逻辑、具体策略、实现机制、工作流程
- 需要分析到"怎么实现的"和"为什么有效"，深度以"能理解其运作原理并判断其价值"为标准
- 关键要素/功能根据情况，配以线框图或示意图辅助理解

### 4. 一般能力简述
- 构成产品/技术的一般性能力，但不是特色或核心的
- 简单论述即可，不展开

### 5. 竞品对比
- 与同类产品/技术的对比分析
- 找出真正的差异化优势，而非表面功能列表对比
- 包括优势、劣势、适用场景的差异

### 6. 洞察与启发
- 洞察：通过分析和调研这个产品项目，有哪些深刻的洞察和认知；
- 启发：你的主人是一个AI的产品经理，作为整个AI产品、业界方向，且他有比较强的vibe coding和产品构建能力的前提下，有哪些可以启发他做落地做产品、demo构建、或者是在工作中能够使用的相关启发；
